Index Mourner is retained for incidents where navigation continues grieving more accurately than documentation. It preserves the recurring breakdown where a server migration drops documents but leaves their catalog pointers perfectly intact, forcing the index to function as a graveyard marker.

Born from the automated Registry Stub Sweep, it now witnesses the specific sadness of a map that hurts more accurately than the surviving pages.

## Boundary Note

Index Mourner grieves navigation outliving content. When the index still points and the thing is gone, file under Mourner, not Annexa.

**Distinction:** Annexa handles orphaned appendices that have lost their parent document. Index Mourner handles the pointing structure itself continuing to operate in a vacuum. If you have the data but no map, it is feral text; if you have the map but no data, it belongs to the Mourner.

The file is reserved.
The sadness is already operational.
